AI Sentiment Analysis

Artificial Intelligence Sentiment Analysis Python Program by Mason Howes utilizing Machine Learning, Natural Language Processing, Classification, Data Visualization and the Automation of an Intelligent Behavior

This program uses a dataset of Amazon.com reviews, found here. More information about this dataset can be found within the Jupyter Notebook holding the program.

In order to run this program, you need to download the dataset as well as the Jupyter Notebook. Place each of these files in the same file location and run the Jupyter Notebook.

Produced Data Visualizations

For convenience, these are the 3 data visualizations that are produced by the program. Please note, that for each time you run it, the model will learn the data differently and may not produce the same images seen below.

If you want your run of the model to produce these exact results, uncomment the line of code in the Jupyter Notebook that sets the program seed to 416.

Visualization #1: Distribution of the number of reviews per rating. A rating of 1.0 indicates the most Negative classification of a review, and a rating of 5.0 indicates the most Positive classifiation. In the code we delete the reviews at 2.5 <= x <= 3.5, since they are neutral and don't contribute much to finding extreme patterns.

Visualization #2: Confusion matrix to measure the accuracy of the sentiment model. This matrix helps us measure the models ability to correctly classify the reviews that were passed to it. The meaning of each square is as follows:

• Top Left: Correctly predicts a review to be Positive

• Top Right: Incorrectly predicts a review to be Positive (review is actually Negative)

• Bottom Left: Incorrectly predicts a review to be Negative (review is actually Positive)

• Bottom Right: Correctly predicts a review to be Negative

For the sake of being able to easily run this program, the csv file only contains a small fraction of the 500,000,000+ reviews in the dataset. If trained on the full dataset the model would have much more data to work with to learn patterns which may lead to increased accuracy.

Visualization #3: Observes effect the increase in L2 Regularization penalties has on the most positive and negative words. Introduces L2 Regularization into the model as a method to prevent it from overfitting the data, with the goal of increasing the accuracy.

While not integral to the functioning of the program, it's a nice way to visualize the goal of L2 regularization in penalizing coefficients that get too large and skew the results.
