ABOUT MASON'S PORTFOLIO

During my time studying for a Data Science degree at the University of Washington, I had the incredible opportunity to expand my programming abilities across multiple coding languages, as well as learned how to hone each of them to accomplish any challenge I am faced with.

Whether the programs I made were for class assignments or as passion projects, I started to realize that compiling them into a central website (even if its just in the form of a portfolio) would be a much better fate for them than if they were left to collect e-dust in my computer's hard drive.

About LetterGrid

About LetterGrid

Originally the only reason for this website's creation, I reached a point with programming where, even if it is relatively simple, I wanted to make something that wasn't just ran out of a script and was presentable.

I had practice with HTML CSS and JavaScript, but most of it was from class assignments that had a planned structure to how the end result should look, but with LetterGrid I finally gave website creation a go from the ground up.

Me and my friends loved playing The Crossword and other word games like Wordle, so I felt like I could try and make something of my own for everyone to try out, and as a result I became even more passionate for programming that I was before.

Alongside the Draw Tool program, this project really pushed my abilities to code something with JavaScript, and while it isn't in its final state since I have more plans for features, I personally find it my favorite thing I've made on this site.

About Speech Transcriber

About Speech Transcriber

As I talk about in the "About" section of the program, this project was originally created in a Jupyter Notebook using Python. However, when trying to port it over to this website, the original library that I used for voice recognition didn't carry over.

This is because of the libraries reliance on Python, and while there are workarounds to run Python code on websites, the library was still not compatible. Therefore, I decided to implement the Web Speech API. I initially wanted to do the entire program from scratch, since I had ample experience with training Machine Learning models on things like handwriting and image recognition, however gaining the number of audio samples required for a model was a task that I didn't have the resources for.

Therefore, I opted to do my best to push the existing APIs to their limits on this program, and even worked things like higher sampling rate, live page position updating and downloadable transcripts. Of the programs on this website, this is the one that I most want to revisit at some point in the future when circumstances are better and attempt to build it truly from the ground up, and see if I can even surpass the existing accuracy of the model.

About Participatory Mapping

About Participatory Mapping

The oldest project on this site at the moment of writing this, Participatory Mapping also showed me the potential of creating websites. Bo Zhao, a GIS professor at the University of Washington, was the one who first taught me how to create websites and through this project I learned more about the potential that web development held.

I had already taken classes that taught SQL (database coding language), but for the most part they were purely queries. Some of them did show me how to work SQL into bigger programs, but this was the first assignment where I really saw the potential it held.

Utilizing the MapBox API for map implementation, Heroku for a dynamic webpage, and pgAdmin4 for database hosting, as well as a mix of HTML, CSS and Javascript for web page creation, and finally SQL for back end development, this project ended up being the program that has the most moving pieces on this site.

About Drawing Tool

About Drawing Tool

Without a doubt, Drawing Tool pushed my understanding of the JavaScript coding language the furthest. Containing over 1,000 lines of code across the HTML, CSS, and JavaScript implementation (with the majority being JavaScript), I believe it to be the most complicated thing that I've programmed on this site.

This program was originally created to see if I could even do it, not as a way to showcase existing knowledge. And while I do think it still has a ways to go, especially with the Mobile version of the menu, I'm still extremely proud of it. Of these projects, it definitely has the most work put into being adaptable to any screen size.

One of the biggest hurdles to overcome in it was both the color selector functionality and the resizing of the drawing functionality, but I believe that I nearly have both of them perfected.

At some point, I do want to go back and clean a lot of it up and even add new features such as pressure sensitivity on mobile devices, and maybe even different brush types, but I think I will save that for if I ever port it to be an actual application instead of a web-based drawing application.
